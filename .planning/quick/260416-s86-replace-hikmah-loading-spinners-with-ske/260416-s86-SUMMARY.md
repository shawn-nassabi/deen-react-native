# Quick Task 260416-s86: Replace Hikmah Loading Spinners with Skeletons Summary

One-liner: Replaced Hikmah tree list and tree detail ActivityIndicator spinners with animated shimmer skeleton cards that mirror the final content shape, matching the treatment already applied to the References page.

## What Changed

### Files Created

1. **`components/hikmah/TreeCardSkeleton.tsx`** (219 lines) — Shimmer skeleton that mirrors `TreeCard`:
   - Outer card (border-radius 16, border-width 1, overflow hidden, shadow matching TreeCard)
   - Title block (60% width, height 20) + 2 summary lines (90%/70%, height 10)
   - Meta row: 2 tag pills (52x18, rounded 12) + count block (60x12)
   - "View Tree" button placeholder (full-width, height 44, rounded 12, bordered)
   - Progress bar placeholder at absolute bottom (height 4, colors.border)
   - Animated.loop shimmer sweep across the card

2. **`components/hikmah/LessonRowSkeleton.tsx`** (154 lines) — Shimmer skeleton row mirroring lesson item:
   - Outer row (border-radius 16, padding 16, flexDirection row, overflow hidden, marginBottom 12)
   - Circular index badge placeholder (32x32, borderRadius 16, borderWidth 1.5)
   - Info column: title block (70% x 14) + est-minutes sub-line (30% x 10)
   - Animated.loop shimmer sweep

### Files Modified

1. **`app/(tabs)/hikmah.tsx`** — Loading branch rewrite:
   - Added `import TreeCardSkeleton from "@/components/hikmah/TreeCardSkeleton"`
   - Removed `ActivityIndicator` from `react-native` imports (no longer used)
   - Loading branch now renders a non-scrollable ScrollView with 3 stacked `<TreeCardSkeleton />` instances below the blur header, followed by the preserved caption "Loading topics..."

2. **`app/hikmah/[treeId].tsx`** — Loading branch rewrite:
   - Added `import LessonRowSkeleton from "@/components/hikmah/LessonRowSkeleton"`
   - Added imports for `useRef, Animated, Easing, Dimensions` from `react-native` and `LinearGradient` from `expo-linear-gradient` (used by the inline hero card skeleton)
   - Removed `ActivityIndicator` from `react-native` imports
   - Loading branch now preserves the full BlurView header (back button + logo + placeholder title block) to prevent header pop-in between loading and loaded states
   - Below the header: inline hero card skeleton (title, 3 summary lines, 2 meta pills, progress bar) + 4 `<LessonRowSkeleton />` instances
   - New bottom-of-file helper `HeroCardSkeletonInner` encapsulates the hero shimmer animation
   - Added supporting styles: `heroSkeletonTitle`, `heroSkeletonLine`, `heroSkeletonMetaRow`, `heroSkeletonMetaPill`, `heroSkeletonProgress`, `shimmerContainer`, `shimmerGradient`

## Key Patterns Reused

- **Canonical shimmer pattern** from `components/references/ReferenceSkeleton.tsx`: `Animated.loop(Animated.timing(..., { duration: 1400, easing: Easing.linear, useNativeDriver: true }))` translating a `LinearGradient` (transparent → highlight → transparent) across an `overflow: "hidden"` card via `transform: [{ translateX }]`.
- **Theme-aware highlight:** `colorScheme === "dark" ? "rgba(255,255,255,0.06)" : "rgba(255,255,255,0.65)"`.
- **Block placeholders** always use `colors.hoverBg` (no hardcoded hex values).
- **React Native Animated** (not Reanimated) per the plan's explicit constraint.
