# Quick Task 260416-s01: Restyle ModalReferenceItem Expanded Metadata Summary

Restyled chat modal reference card expanded view to render a single italic citation paragraph (branched per hadith vs Quran) in place of the labeled metadata grid, mirroring the standalone `ReferenceItem.tsx` pattern from quick task 260416-rqn.

## What Was Done

### Task 1: Replace metadataGrid with branched italic citation paragraph

**File modified:** `components/chat/ModalReferenceItem.tsx`

Changes:
- Added `buildCitation()` helper above the return statement. Branches on `isQuran` and emits the correct ordered citation parts joined by ` · `, skipping empty/N/A/unspecified values:
  - **Hadith order:** `collection · author · Hadith #N · Book N: Title · Ch. N: Title · Vol. N · reference · Graded {grade}`
  - **Quran order:** `surah_name · Verses {x-y} · {tafsir title} · author · collection · Vol. N`
- Replaced the `<View style={styles.metadataGrid}>` block (which previously contained the `isQuran` ternary with labeled fields) with a single IIFE-wrapped `<Text style={[styles.citationText, { color: colors.textSecondary }]}>` that only renders when the citation is non-empty.
- Removed the hadith `<Text style={styles.textLabel}>Text</Text>` label above the hadith English/Arabic body. The textSection divider + body content is unchanged.
- Preserved the Quran "Translation" and "Tafsir" uppercase labels — required because the Quran branch renders two distinct text bodies.
- Removed the `renderField` helper (no longer used by either branch).
- Removed unused styles: `metadataGrid`, `field`, `fieldLabel`, `fieldValue`.
- Kept `textLabel` style (still consumed by Quran Translation/Tafsir labels).
- Added new `citationText` style: `{ fontSize: 13, lineHeight: 20, fontStyle: "italic", marginBottom: 16 }` — matches `ReferenceItem.tsx` `styles.citationText`.

### Task 2: Visual verification — chat modal reference cards

User visually verified both hadith and Quran reference cards in the chat modal and confirmed:
- Citation paragraphs render in the correct branch-specific order with ` · ` separators
- No "Text" label on hadith; "Translation" and "Tafsir" labels still present on Quran
- Empty/N/A/unspecified fields are cleanly skipped
- Condensed view, copy behavior, chevron animation, and footer are unchanged

User responded "approved".
