# Phase 1.3: Password Reset Flow - Context

**Gathered:** 2026-04-11
**Status:** Ready for planning

<domain>
## Phase Boundary

Deliver a complete "forgot password" UX: a dedicated Forgot Password screen where users enter their email and receive a reset link, and a New Password screen reached via deep link where they set and confirm a new password and are automatically signed in. No browser redirect — the full flow stays in-app.

Screens being added:
- `app/forgot-password.tsx` — email entry + inline confirmation state
- `app/reset-password.tsx` — new password form; handles PKCE code exchange from deep link

Config change:
- `app.json` — `deenreactnative://reset-password` linked route registered (MIG-11)

Auth guard update:
- `app/_layout.tsx` — `isOnAuthScreen` extended to include `"forgot-password"` and `"reset-password"` so deep link lands without redirect to `/login`

Password reset (AUTH-06, AUTH-07), deep link (MIG-11), and both UI screens (UI-03, UI-04) are in scope. Cognito package removal and user ID replacement are Phase 1.4.

</domain>

<decisions>
## Implementation Decisions

### Forgot Password Navigation
- **D-01:** Separate screen — `app/forgot-password.tsx` alongside `app/login.tsx` and `app/signup.tsx`. Consistent with Phase 1.2 D-05 flat screen structure. `router.push('/forgot-password')` from the login screen's "Forgot password?" link.
- **D-02:** Screen uses the same elevated card layout as login/signup (Phase 1.2 D-01): logo above the card, card contains heading, email field, button, and "← Back to sign in" link.

### Post-Email Confirmation UX
- **D-03:** Inline confirmation on the same screen — after submitting the email, the email input, button, and heading are replaced by a "Check your email" confirmation state within the same card. No navigation. The submitted email address is shown in the confirmation message (e.g. "We've sent a reset link to user@example.com"). "← Back to sign in" link remains accessible.
- **D-04:** Confirmation copy: heading "Check your email", body "We've sent a reset link to [email]. Check your inbox and junk folder."

### Supabase Reset Mode (Deep Link)
- **D-05:** PKCE flow — Supabase dashboard "Use PKCE flow" is ON. Reset links arrive as `deenreactnative://reset-password?code=abc123`. The app calls `supabase.auth.exchangeCodeForSession(code)` to obtain a session.
- **D-06:** The New Password screen (`app/reset-password.tsx`) handles the code exchange itself. On mount, it reads the `code` URL param via `useLocalSearchParams`, shows a spinner while `exchangeCodeForSession()` runs, then reveals the new-password form once the session is confirmed. No separate processing route.
- **D-07:** `supabase.auth.updateUser({ password: newPassword })` is called with the active session to set the new password. After success, the session is already live — `_layout.tsx` redirect logic fires and routes to `/(tabs)` automatically.

### New Password Form
- **D-08:** Two fields — "New password" and "Confirm new password". Required by UI-04. Same bordered input style, emerald focus highlight, show/hide toggle.
- **D-09:** Client-side validation before submit: both fields must match; show inline error "Passwords don't match" if they differ. No submission attempt until they match.

### Auth Guard Extension
- **D-10:** `isOnAuthScreen` in `app/_layout.tsx` extended to: `segments?.[0] === "login" || segments?.[0] === "signup" || segments?.[0] === "forgot-password" || segments?.[0] === "reset-password"`. This allows the deep link to land on `reset-password` without the unauthenticated redirect firing.

### Route Registration
- **D-11:** Both new screens registered in `app/_layout.tsx` `<Stack>`:
  - `<Stack.Screen name="forgot-password" options={{ headerShown: false }} />`
  - `<Stack.Screen name="reset-password" options={{ headerShown: false }} />`
- **D-12:** `app.json` — `deenreactnative://reset-password` scheme entry required for deep link. This is already partially satisfied by the existing `scheme: "deenreactnative"` — Expo Router maps any `deenreactnative://reset-password?...` URL to `app/reset-password.tsx` automatically via file-based routing. No additional `app.json` plugin config needed beyond confirming the scheme is set. MIG-11 is satisfied when the file exists.

### Claude's Discretion
- Exact card padding, border radius, and spacing — mirror login.tsx / signup.tsx values.
- Error message strings for `exchangeCodeForSession` failures (expired link, already used link) — clear, friendly, include "Request a new reset link" prompt.
- Whether the reset-password screen handles missing/invalid `code` param gracefully (no code → show error state, not crash).
- Loading spinner style on the reset-password screen — match the busy spinner pattern from login.tsx.

</decisions>

## Existing Code Insights

### Reusable Assets
- `app/login.tsx` — Full elevated card pattern. The forgot-password screen is essentially a login card with one field and different copy. Mirror it directly.
- `app/signup.tsx` — Same structure; also shows `mapSignUpError()` pattern for error message mapping.
- `hooks/useAuth.tsx` — Exposes `useAuth()`. Reset flow calls `supabase` directly (not via `useAuth`) for `exchangeCodeForSession` and `updateUser`, since these are one-off operations not part of the hook's managed auth state.
- `utils/supabase.ts` — The singleton `supabase` client. `reset-password.tsx` imports it directly.

### Established Patterns
- `StyleSheet.create({})` at bottom of file, named `styles`.
- `useState` for `busy`, `error`, form field values, and `showPassword` toggle.
- Inline `ThemedText` error with `{ color: "#ef4444" }` below the button.
- `ActivityIndicator` inside button when `busy` is true.
- `ThemedView` / `ThemedText` / `Colors[colorScheme]` throughout.
- `useLocalSearchParams()` from `expo-router` to read deep link URL params.

### Integration Points
- `app/_layout.tsx` — Two changes needed: extend `isOnAuthScreen` guard, add two `<Stack.Screen>` entries.
- `app/login.tsx` — One change: "Forgot password?" `TouchableOpacity`'s `onPress` wired to `router.push('/forgot-password')` (currently `onPress={() => {}}`).
- `supabase.auth.resetPasswordForEmail(email, { redirectTo: 'deenreactnative://reset-password' })` — called in forgot-password.tsx.
- `supabase.auth.exchangeCodeForSession(code)` — called in reset-password.tsx on mount.
- `supabase.auth.updateUser({ password: newPassword })` — called in reset-password.tsx on submit.

### Testing Constraint
- Custom scheme deep links (`deenreactnative://`) do not work in Expo Go. Phase 1.3 must be tested on a dev-client build only.
