## Decisions Made

- No `router.replace("/(tabs)")` after `updateUser` ŌĆö the `USER_UPDATED` event with valid session triggers `onAuthStateChange` in `useAuth.tsx`, which sets `status="signedIn"`, which triggers the `_layout.tsx` `useEffect` redirect. Adding a manual replace would race with the automatic one.
- Mount guard is mandatory for `exchangeCodeForSession`: without it, React Strict Mode double-invokes the `useEffect` in dev builds, consuming the PKCE verifier on the first call and producing `AuthPKCECodeVerifierMissingError` on the second.
