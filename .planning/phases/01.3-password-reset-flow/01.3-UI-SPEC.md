## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `colors.background` — light `#f8f9fa`, dark `#0a0b09` | Full-screen ThemedView background |
| Secondary (30%) | `colors.panel` — light `#ffffff`, dark `#111211` | Card background; `colors.panel2` (light `#f3f4f6`, dark `#1a1a1a`) for input fill |
| Accent (10%) | `colors.primary` — `#5bc1a1` (emerald) | See reserved-for list below |
| Destructive | `#ef4444` | Inline error text only |

Accent reserved for:
- Focused input border (`borderColor: emailFocused ? colors.primary : colors.border`)
- Primary CTA button background (`backgroundColor: colors.primary`)
- "Back to sign in" link text on confirmation state (`color: colors.primary`)
- Mail icon in confirmation state (`color={colors.primary}`)
- Confirmation link text (`color: colors.primary`)

NOT used on:
- Card borders (no card border — shadow only)
- Muted/secondary links (use `colors.muted` — light `#6b7280`, dark `#6b7280`)

Input border at rest: `colors.border` — light `#d1d5db`, dark `#2a2a2a`.
Placeholder text: `colors.muted` — `#6b7280` both modes.

Source: constants/theme.ts, login.tsx lines 78–84, 107–112.

## Screen Contracts

### Screen 1: `app/forgot-password.tsx`

**Two internal states within the same screen — no navigation between them.**

#### State A: Email Entry Form

Layout (top to bottom, centered via ScrollView):
1. Logo — `deen-logo-with-text.png`, width 100% maxWidth 320, height 120, `marginBottom: 24`
2. Elevated card — width 100%, `borderRadius: 20`, `padding: 24`, `paddingBottom: 32`, `shadowOpacity: 0.10`, `shadowRadius: 12`, `elevation: 6`
   - Heading: "Reset your password" — 20px / 600
   - Email input — `marginTop: 16`, `borderRadius: 12`, `paddingHorizontal: 16`, `paddingVertical: 14`, fontSize 14, `backgroundColor: colors.panel2`, border 1px `colors.border` at rest / `colors.primary` on focus
   - Inline error text (conditional) — `color: "#ef4444"`, 13px, centered, `marginTop: 8`
   - CTA button — "Send reset email", `marginTop: 24`, `borderRadius: 14`, `paddingVertical: 14`, `backgroundColor: colors.primary`, label 16px / 600 / `#fff`
   - Busy state inside button — `ActivityIndicator color="#fff"` + label "Sending…" in a `flexDirection: "row"` with `gap: 8`; button `opacity: 0.75` when busy
   - "Back to sign in" link — `marginTop: 16`, `fontSize: 14`, `fontWeight: "500"`, `color: colors.muted`, `paddingVertical: 10`; `router.back()` or `router.replace("/login")` if no history

#### State B: Confirmation (replaces card contents after successful email dispatch)

Card contents swapped (card container unchanged — same shadow, borderRadius, padding):
1. Mail icon — `Ionicons name="mail-outline"` size 40, `color={colors.primary}`, centered, `marginBottom: 12`, `marginTop: 4`
2. Heading: "Check your email" — 20px / 600, `textAlign: "center"`
3. Body copy — 14px / 400, `color: colors.muted`, `textAlign: "center"`, `marginTop: 8`, `lineHeight: 22`:
   > "We've sent a reset link to **[email]**. Check your inbox and junk folder."
   - The email address is rendered as an inline `ThemedText` with `fontWeight: "600"` and `color: colors.text` (not muted)
4. "Back to sign in" link — same as State A link style but `color: colors.primary`

State transition trigger: `supabase.auth.resetPasswordForEmail()` resolves without error. State variable `sent: boolean` controls the swap.

### Screen 2: `app/reset-password.tsx`

**Three internal states: loading, form, and success-redirect (auto).**

#### State A: Loading (code exchange in progress)

Full-screen centered spinner within the same card layout:
- Logo above card (same as all auth screens)
- Card contains:
  - `ActivityIndicator size="large" color={colors.primary}` centered
  - Body text below spinner: "Verifying reset link…" — 14px / 400, `color: colors.muted`, `textAlign: "center"`, `marginTop: 8`

#### State B: Error (code exchange failed — expired, already used, or missing code param)

Card contains:
- Heading: "Link expired" — 20px / 600, `textAlign: "center"`
- Body: "This reset link has expired or has already been used. Request a new one from the sign in screen." — 14px / 400, `color: colors.muted`, `textAlign: "center"`, `marginTop: 8`, `lineHeight: 22`
- "Back to sign in" link — `color: colors.primary`, 14px / 500, `paddingVertical: 10`; `router.replace("/login")`

Missing `code` param (no query string on deep link): treat identically to expired link — show the same error state rather than crash.

#### State C: New Password Form (code exchange succeeded)

Card contains:
1. Heading: "Set new password" — 20px / 600
2. "New password" input — `marginTop: 16`, same input style as login.tsx; `secureTextEntry={!showPassword1}`; show/hide toggle via `Ionicons eye-outline / eye-off-outline` size 20, `color={colors.muted}`; `paddingRight: 48` when toggle present
3. "Confirm new password" input — `marginTop: 16`, same style; separate show/hide toggle; `secureTextEntry={!showPassword2}`; `returnKeyType: "go"`
4. Inline validation error (conditional) — shown below both inputs if passwords do not match on submit attempt: "Passwords don't match." — `color: "#ef4444"`, 13px, centered, `marginTop: 8`
5. Inline API error (conditional) — shown if `updateUser()` fails: see error copy table below
6. CTA button — "Save password", `marginTop: 24`, same button style
   - Busy state: `ActivityIndicator color="#fff"` + "Saving…" label
7. No "back" link — user arrived via deep link; back navigation is not meaningful here

#### State D: Success (auto-redirect)

No explicit success state rendered. After `supabase.auth.updateUser()` succeeds:
- Session is live in Supabase client
- `_layout.tsx` `onAuthStateChange` fires and redirects to `/(tabs)` automatically
- No manual `router.replace` needed in reset-password.tsx

## Interaction Contracts

### Focus Management
- Forgot Password: email input auto-focuses on mount (`autoFocus={true}`)
- Reset Password form: "New password" input auto-focuses after code exchange succeeds (after state transitions from loading to form)
- "New password" → "Confirm new password": `returnKeyType="next"`, `onSubmitEditing` moves focus via ref
- "Confirm new password": `returnKeyType="go"`, `onSubmitEditing` triggers submit

### Validation Timing
- Password match validation: triggered on submit only, not on change (prevent premature errors)
- Empty field check: triggered on submit; show "Please fill in both password fields." error if either is empty
- Email empty check on forgot-password: triggered on submit; show "Please enter your email address." if blank

### Busy / Disabled States
- `disabled={busy}` on all buttons while async in flight
- Button `opacity: 0.75` when `busy` is `true`
- Inputs remain editable during busy (consistent with login.tsx)

### Error Clearing
- `setError(null)` at the start of every submit handler (before `setBusy(true)`) — consistent with login.tsx pattern

### Keyboard Behavior
- `keyboardShouldPersistTaps="handled"` on ScrollView — consistent with login.tsx and signup.tsx

### Accessibility
- Show/hide toggle `accessibilityLabel`: `showPassword ? "Hide password" : "Show password"` — each toggle independently labelled
- Mail icon in confirmation state: decorative; no accessibilityLabel needed (heading provides context)
