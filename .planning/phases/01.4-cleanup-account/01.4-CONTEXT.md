**Gathered:** 2026-04-13

<domain>
## Phase Boundary

Strip all remaining Cognito artefacts from the codebase, replace user ID references with the Supabase UUID, and ship a working account deletion flow. Delivers:

- `expo-auth-session` and `expo-web-browser` removed from `package.json` and `app.json` (MIG-09)
- `components/external-link.tsx` updated to use `Linking.openURL` instead of `openBrowserAsync`
- `app/auth.tsx` (dead OAuth callback screen) deleted (MIG-08)
- `user?.email || user?.sub` replaced with `user.id` in all three files (MIG-05)
- `AuthUser.sub` alias removed from the type after all call sites are updated
- `deleteAccount()` added to `utils/api.ts`; full Delete Account UI added to `app/settings.tsx` (ACC-01)

Password reset flow tech debt (PKCE `verifyOtp` fix) is NOT in scope — tracked separately in CONCERNS.md.

</domain>

<decisions>
## Implementation Decisions

### expo-web-browser Removal (MIG-09)
- **D-01:** Replace `openBrowserAsync` in `components/external-link.tsx` with React Native's built-in `Linking.openURL`. No new dependency. Opens the system browser instead of an in-app sheet (minor UX difference, acceptable).
- **D-02:** After fixing `external-link.tsx`, remove BOTH `expo-auth-session` AND `expo-web-browser` from `package.json` dependencies. Also remove `"expo-web-browser"` from the `plugins` array in `app.json`. MIG-09 fully satisfied.
- **D-03:** Run `pod install` after package removal to clean native modules.

### Dead File Deletion (MIG-08)
- **D-04:** Delete `app/auth.tsx` — the Cognito OAuth deep-link callback screen. No `Stack.Screen name="auth"` exists in `app/_layout.tsx` (already absent), so no layout changes needed. File deletion is the only action.

### User ID Replacement (MIG-05)
- **D-05:** Replace `const userId = user?.email || user?.sub` with `const userId = user?.id` in:
  - `app/(tabs)/hikmah.tsx` (line 41)
  - `app/hikmah/lesson/[lessonId].tsx` (line 47)
  - `components/hikmah/ElaborationModal.tsx` (line 82)
  
  `AuthUser.id` is already the Supabase UUID field from Phase 1.2 D-09 — this is a mechanical substitution.
- **D-06:** `components/hikmah/LessonQuizPage.tsx` takes `userId` as a prop — fixing the parent (lessonId.tsx) automatically fixes the prop value passed down. No change needed in `LessonQuizPage.tsx` itself.
- **D-07:** `app/settings.tsx` line 139 shows `user?.sub` as a display value — update to `user?.id` for consistency.

### AuthUser.sub Alias Removal
- **D-08:** Remove `sub?: string` from the `AuthUser` type in `hooks/useAuth.tsx` after all call sites have been updated by D-05/D-07. This was explicitly deferred to Phase 1.4 in Phase 1.2 D-09. Removing it makes any lingering `user.sub` references a TypeScript compile error — a safety net.

### Account Deletion (ACC-01)
- **D-09:** Add `deleteAccount()` to `utils/api.ts` — calls `DELETE /account/me` with the Supabase Bearer token (via `getValidAccessToken()`). Returns `void` on 204 success; throws a descriptive `Error` on failure. Follow existing API function patterns (file-level JSDoc, error throw on non-2xx).
- **D-10:** Add a "Delete Account" button to `app/settings.tsx` in a danger zone at the bottom of the screen, below the existing Sign Out button. Visually separated (section separator or margin). Styled in a destructive/red color using theme tokens.
- **D-11:** Confirmation via `Alert.alert()` (React Native native alert). Title: "Delete Account". Message warns the action is permanent. Buttons: "Cancel" (default, safe) and "Delete" (destructive style). No submission until confirmed.
- **D-12:** On successful deletion — `deleteAccount()` completes → call `signOut()` from `useAuth` → `_layout.tsx` auth guard redirects to `/login` automatically. No manual `router.replace` needed (same pattern as Phase 1.3 D-07 post-updateUser flow).
- **D-13:** Error handling: wrap in try/catch; show `Alert.alert("Error", errorMessage)` on failure. User stays on settings screen and can retry.

### Claude's Discretion
- Exact red/destructive color for the Delete Account button — use `colors.error` or `#e53935` if no error token exists in `constants/theme.ts`. Check theme tokens before hardcoding.
- Loading state for the delete action — show an `ActivityIndicator` while the API call is in flight, disable the button to prevent double-tap.
- Section separator style between Sign Out and Delete Account — match the existing settings section separator pattern.

</decisions>

## Existing Code Insights

### Reusable Assets
- `components/external-link.tsx` — currently imports `openBrowserAsync, WebBrowserPresentationStyle` from `expo-web-browser`. D-01 replaces this with `import { Linking } from 'react-native'` and swaps the call to `Linking.openURL(href)`.
- `utils/api.ts` — `deleteAccount()` should follow the existing pattern of other void-return API functions (e.g., the `signOut`-adjacent utilities). Use `getValidAccessToken()` for the Bearer token, same as all other API calls.
- `app/settings.tsx` — already has Sign Out button and a section-based layout. The danger zone goes below Sign Out, following the same `TouchableOpacity` + `StyleSheet` pattern already present.
- `hooks/useAuth.tsx` — `AuthUser` type defined here. `sub?: string` alias was added in Phase 1.2 D-09; remove it alongside the D-05/D-07 user ID replacements.

### Established Patterns
- API functions: file-level JSDoc block, `getValidAccessToken()` for Bearer, throw descriptive `Error` on non-2xx.
- Error display in settings: `Alert.alert()` — already used in settings for sign-out confirmation in other apps; consistent approach here.
- Destructive actions: `Alert.alert` with Cancel (default) / Destructive button — standard React Native pattern for irreversible actions.

### Integration Points
- `app/_layout.tsx` — no changes needed for auth route removal (no `Stack.Screen name="auth"` entry exists).
- `app.json` — remove `"expo-web-browser"` from `plugins` array. No other Cognito-specific plugin entries remain.
- `package.json` — remove `expo-auth-session` and `expo-web-browser` from `dependencies`.
