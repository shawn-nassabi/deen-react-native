## Decisions Made

**D-02 LargeSecureStore Chunking (not D-01 AES-256):**
The plan notes explicitly chose numeric-suffix chunking over the AES-256 approach from the official Supabase React Native guide. Rationale: zero new dependencies (only `expo-secure-store` already installed), no crypto dependency risk on Hermes, simpler and more predictable. AES-256 approach requires `aes-js` + `react-native-get-random-values` — avoided entirely.

**D-08 Cognito Fields Removed in Phase 1.1:**
Per decision D-08, all five Cognito fields (COGNITO_DOMAIN, COGNITO_CLIENT_ID, COGNITO_ISSUER, COGNITO_SCOPES, AUTH_REDIRECT_URI) were fully removed from CONFIG in this phase. To keep the existing Cognito auth flow working, utils/auth.ts was updated to read these values directly from `process.env.EXPO_PUBLIC_COGNITO_*`. This decouples auth.ts from CONFIG for Cognito values and keeps CONFIG clean for the migration path forward.

**requireEnv() Fail-Fast Pattern:**
Used `requireEnv()` for Supabase env vars rather than `|| defaultValue`. This ensures the app throws a clear error at startup if Supabase credentials are missing, rather than silently using empty/wrong values. The `.env` placeholder file provides values so the guard does not trigger during Phase 1.1 development.
