## Dev Config Defaults

| Option | Description | Selected |
|--------|-------------|----------|
| Strict env-vars only | No hardcoded fallbacks. Missing env vars → clear error/warning. | ✓ |
| Hardcoded dev fallbacks | Mirror Cognito pattern — real Supabase URL and key as fallbacks. Convenient but keys in git history. | |
| Placeholder fallbacks | Empty string fallbacks — fail gracefully with recognizable error rather than crash. | |

**User's choice:** Strict env-vars only.
**Notes:** User also noted that the latest Supabase pattern uses "Publishable key" (sb_publishable_...) not the legacy anon key JWT. Docs: https://supabase.com/docs/guides/api/api-keys
