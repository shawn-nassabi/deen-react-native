# Phase 01.2 — UI Design Contract

> Visual and interaction contract for Phase 01.2: Auth Core + Login UI.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom React Native StyleSheet |
| Preset | not applicable |
| Component library | none — custom primitives (`ThemedView`, `ThemedText`, `TouchableOpacity`, `TextInput`) |
| Icon library | `@expo/vector-icons` (Ionicons) — already installed |
| Font | Montserrat (loaded via `@expo-google-fonts/montserrat` in `app/_layout.tsx`) |

Note: This is a React Native / Expo project. shadcn is a React web component library and is not applicable here. The existing design system is `constants/theme.ts` (`Colors` token map, `Fonts` platform stack) + `StyleSheet.create({})` per-component styles.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (e.g. gap between `ActivityIndicator` and button label) |
| sm | 8px | Inter-element spacing within card (e.g. between input border focus ring and label) |
| md | 16px | Default element spacing — between card children, input vertical padding |
| lg | 24px | Card internal padding (horizontal + top); spacing between logo and card |
| xl | 32px | Card bottom padding; spacing between sections |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets for `TouchableOpacity` links (Forgot password, Sign up): minimum 44px tall — achieved via `paddingVertical: 10` on link rows (source: iOS HIG tap target minimum)
- Button `paddingVertical`: 14px — carried forward from existing `login.tsx`
- Logo height: 120px (fixed, source: existing `login.tsx`)
- Card `borderRadius`: 20px (elevated card; existing button uses 14px — card is larger, uses 20px)

## Typography

All sizes in React Native density-independent points. Montserrat loaded globally.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Error text | 13pt | 400 (regular) | — |
| Body (input placeholder, helper text) | 14pt | 400 (regular) | 1.5 |
| Label (input label, link text) | 14pt | 400 (regular) | 1.4 |
| Button text | 16pt | 600 (semibold) | 1.25 |
| Heading (card title) | 20pt | 600 (semibold) | 1.2 |

Source: existing `login.tsx` uses `fontSize: 14` for subtitle, `fontSize: 16` / `fontWeight: "600"` for button text. Heading (20pt) and body/label (14pt) fill the gap to give 3 distinct levels. 4 sizes, 2 weights.

## Color

All color values sourced from `Colors[colorScheme]` token map in `constants/theme.ts`. Hardcoded hex allowed only for `#fff`, `#000`, and the universal error red `#ef4444`.

| Role | Light token → value | Dark token → value | Usage |
|------|---------------------|--------------------|-------|
| Dominant (60%) | `colors.background` → `#f8f9fa` | `colors.background` → `#0a0b09` | Full-screen background behind logo and card |
| Secondary (30%) | `colors.panel` → `#ffffff` | `colors.panel` → `#111211` | Card surface (elevated container holding inputs and button) |
| Input surface | `colors.panel2` → `#f3f4f6` | `colors.panel2` → `#1a1a1a` | `TextInput` background fill inside bordered input fields |
| Accent (10%) | `colors.primary` → `#5bc1a1` | `colors.primary` → `#5bc1a1` | See reserved list below |
| Border (default) | `colors.border` → `#d1d5db` | `colors.border` → `#2a2a2a` | Input field default border |
| Destructive | `#ef4444` (universal) | `#ef4444` (universal) | Inline error message text only |

Accent (`#5bc1a1`) reserved for:
1. Primary button background (`Sign In` / `Sign Up` button fill)
2. Focused `TextInput` border highlight (replaces `colors.border` when field is focused)

Accent is NOT used for: card borders, links, helper text, logo tint, or the card shadow.

### Layout Structure (both screens)

```
ThemedView (flex: 1, background: colors.background)
  ScrollView (keyboardShouldPersistTaps: 'handled')
    View (content wrapper, alignItems: center, paddingHorizontal: 24)
      Image (logo, width: 100%, maxWidth: 320, height: 120, marginBottom: 24)
      View (card, background: colors.panel, borderRadius: 20, padding: 24, shadow)
        ThemedText (heading: "Sign in to continue" / "Create account", fontSize: 20, weight 600)
        View (email input group, marginTop: 16)
          TextInput (email field, bordered, keyboardType: email-address)
        View (password input group, marginTop: 16)
          TextInput (password field, secureTextEntry, show/hide toggle)
        ThemedText (error message, color: #ef4444, fontSize: 13, marginTop: 8) — conditional
        TouchableOpacity (primary button, marginTop: 24, background: colors.primary)
          [busy] ActivityIndicator + ThemedText "Signing in…" / "Creating account…"
          [idle] ThemedText "Sign in" / "Sign up"
        View (links row, marginTop: 16)
          TouchableOpacity "Forgot password?" — login screen only
          TouchableOpacity "Don't have an account? Sign up" / "Already have an account? Sign in"
```

Note: `ScrollView` wrapping ensures the card remains accessible when the software keyboard appears on small devices. This replaces the plain `View` content wrapper in the current `login.tsx`.

### Password Show/Hide Toggle

Add a show/hide toggle to both password fields (CONTEXT.md: Claude's discretion). Implementation:
- `useState<boolean>(false)` for `showPassword`
- Toggle button: `TouchableOpacity` positioned `absolute` inside the input container, right-aligned
- Icon: `Ionicons` — `eye-outline` (hidden) / `eye-off-outline` (visible), size 20, color `colors.muted`
- Toggling sets `secureTextEntry={!showPassword}`
- `accessibilityLabel`: `"Show password"` when hidden / `"Hide password"` when visible

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Tap targets minimum 44pt | All `TouchableOpacity` links use `paddingVertical: 10` minimum |
| Email field keyboard type | `keyboardType="email-address"`, `autoCapitalize="none"`, `autoCorrect={false}` |
| Password field | `secureTextEntry={true}` (toggled by show/hide), `autoCapitalize="none"`, `autoCorrect={false}` |
| Return key | Email field: `returnKeyType="next"`, focus jumps to password; Password field: `returnKeyType="go"`, triggers form submit |
| Disabled state | Button sets `disabled={busy}` — prevents double submission |
