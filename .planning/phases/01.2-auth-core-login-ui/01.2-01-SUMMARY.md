## Decisions Made
- The `onAuthStateChange` callback must be synchronous — the Supabase SDK holds an exclusive lock during the call; any `await` inside (e.g., calling `getSession`) will cause a deadlock. State setters are the only calls made inside the callback.
- INITIAL_SESSION fires immediately during `createClient` initialization — the subscription must be registered synchronously (not inside `async`) to capture it. This is why the MIG-07 cleanup is fire-and-forget: we cannot await it before registering the listener.
- `AuthUser.sub` preserved as backward compat alias for `session.user.id` — multiple screens use `user?.sub` and will be cleaned up in Phase 1.4.
- `signOut(opts?)` signature kept for backward compat with `settings.tsx` call site; `opts.global` is silently ignored since Supabase has no equivalent.
